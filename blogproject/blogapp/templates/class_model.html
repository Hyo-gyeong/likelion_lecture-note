데이터베이스 = 정보 저장 공간 - model이 데이터 베이스를 다뤄줌 따라서 model에 가공하고자 하는 데이터, 가공되었으면하는 데이터의 종류는 무엇인지 또 어떤 정보가 처리되기를 원하는지 class로 정의해주기! 예_ 짧은글 + 시간 + 긴 글 이런 뭉터기로 생긴 하나의 데이터 형식을 처리하기로 model에 class로 정의해주었기때문에 블로그 글쓰기 같은 형식이 나타날 수 있는 것! model.py = 데이터를 만드는 공장 그 속 class = 데이터 틀 그 속에 data(객체)있음 model = 데이터베이스를 뒤적뒤적,,,, ---->>>(데이터베이스의 종류가 다양함!)따라서 장고와 완전 별개이며 여러개가 있을 수 있음! (settings.py에서 등록 가능) ---->>>야아아아아아 나 방금 models.py안에 이렇게 생긴 데이터를 처리하기로 했어 하구 database한테 알려줘야함 그 명령어가 python manage.py makemigrations(파일만들기)/ python manage.py migrate(실재로 데이터베이스에 만들걸 적용하는 과정) -admin계정 만들어서 추가할 수 있도록 해야지 명령어 = python manage.py createsuperuser 그리고 우리가 방금 만든 데이터를 처리하겠다고 알려줘야하기 때문에 model안 class를 admin.py에 들어가서 정의해주어야 함 한 줄 요약:::::::::::::: 데이터베이스에 어떻게 생긴 데이터를 넣을지 정의하고, 거기에 admin권한으로 데이터 집어넣기(=글쓰기)
